Методы

Методы это события, которые сигнализируют нативному приложению Telegram о необходимости выполнения какого-либо действия. Они всегда вызываются приложением TWA.

Web​

Так как веб-версия Telegram отображает приложение TWA в HTML-теге <iframe/>, оно использует стандартный способ общения между двумя фреймами - при помощи отправки сообщений посредством вызова функции window.parent.postMessage.

Первым параметром необходимо передавать объект JSON приведенный к строке. Этот объект должен иметь следующий интерфейс:

interface MessageJSON {
  eventType: string;
  eventData: any;
}

Вторым параметром является строка, которая описывает допустимый origin родительского фрейма.

подсказка

Мы рекомендуем избегать использования значения * ввиду того, что это не безопасно. Ваше приложение может быть обёрнуто во фрейм злоумышленника, чего оно не узнает продолжив общаться с ним ввиду игнорирования родительского origin.

Как значение по умолчанию, вы можете использовать https://web.telegram.org.

Как могло быть подмечено ранее, название каждого метода описывается при помощи свойства eventType и связанных параметров, находящихся в свойстве eventData:

Пример вызова метода
window.parent.postMessage(JSON.stringify({
  eventType: 'web_app_setup_back_button',
  eventData: {is_visible: true},
}), 'https://web.telegram.org');

Код выше сообщит нативному приложению о том, что необходимо показать Кнопку Назад.

Desktop и mobile​

В отличие от web-версии, desktop- и mobile-версии используют чуть более необычный способ вызова методов. При помощи вставки своего собственного JavaScript кода в веб-среду, они создают специальную глобальную функцию window.TelegramWebviewProxy.postEvent, принимающую 2 аргумента. Первый аргументом является наименование вызываемого метода, вторым - объектом со связанными параметрами, приведенным к строке:

Пример вызова метода
window.TelegramWebviewProxy.postEvent('web_app_setup_back_button', JSON.stringify({
  is_visible: true
}));

Windows Phone​

Нативное приложение Telegram на Windows Phone создает функцию window.external.notify. В качестве первого параметра необходимо передавать приведенный к строке объект, описанный в разделе для web-версии Telegram:

Пример вызова метода
window.external.notify(JSON.stringify({
  eventType: 'web_app_setup_back_button',
  eventData: {is_visible: true},
}))

Доступные методы​

Эта секция содержит список доступных для вызова методов, включая их наименование, описание и параметры. Заголовок секции указывает на то, начиная с какой версии содержащиеся внутри секции методы становятся доступными.

предупреждение

В случае, если приложение TWA пытается вызвать метод, неподдерживаемый текущей версией Telegram Web Apps, ничего не произойдет. Нативное приложение никак не сигнализирует о том, что вызван неизвестный метод. Это необходимо учитывать в случае кажущейся некорректной работы метода.

v6.0​

iframe_ready​

Уведомляет родительский фрейм о том, что текущий фрейм готов к работе. Этот метод используется только web-версией Telegram и как результат вызова, будет создано событие set_custom_style.

web_app_close​

Закрывает приложение TWA.

web_app_data_send​

Отправляет сервисное сообщение боту Telegram. Для получения большей информации касательно отправляемых данных, обратите внимание на поле web_app_data в классе Message.

После вызова метода приложение TWA будет закрыто.

ПолеТипОписание
datastringДанные для отправки боту. Максимальный размер 4096 байт.

web_app_expand​

Максимизирует viewport приложения TWA.

web_app_open_link​

Открывает ссылку в стандартном браузере приложения.

ПолеТипОписание
urlstringURL для открытия. Значение должно представлять полный путь с указанием https протокола.

web_app_ready​

Уведомляет нативное приложение Telegram о готовности текущего приложения TWA к отображению. Вызов этого метода приведёт к удалению блока с лоадером, располагаемым поверх приложения TWA.

web_app_request_theme​

Запрашивает текущую тему от нативного приложения Telegram. В качестве результата будет создано событие theme_changed.

web_app_request_viewport​

Запрашивает информацию о текущем viewport-е от нативного приложения Telegram. В качестве результата будет создано событие viewport_changed.

web_app_setup_main_button​

Обновляет настройки Главной Кнопки.

ПолеТипОписание
is_visiblebooleanОпциональное. Отобразить кнопку.
is_activebooleanОпциональное. Сделать кнопку активной.
is_progress_visiblebooleanОпциональное. Отобразить лоадер внутри кнопки. Используйте это свойство в случае, если нажатие на кнопку вызывает какую-либо длительную операцию и пользователь должен быть об этом осведомлён.
textstringОпциональное. Текст внутри кнопки.
colorstringОпциональное. Фоновый цвет кнопки в формате #RRGGBB.
text_colorstringОпциональное. Цвет текста кнопки в формате #RRGGBB.

web_app_setup_closing_behavior​

Устанавливает поведение закрытия приложения.

ПолеТипОписание
need_confirmationbooleanНеобходимо ли запрашивать подтверждение закрытия у пользователя.

v6.1​

web_app_open_tg_link​

Открывает внутреннюю ссылку Telegram.

После вызова метода приложение TWA будет закрыто.

ПолеТипОписание
path_fullstringПуть для открытия. Должно быть значением из ссылки следующего формата: https://t.me/{path_full}. Значение может также содержать query-параметры.

web_app_open_invoice​

Открывает платежный счёт при помощи указанного идентификатора. Более подробную информацию о счетах можно найти в официальной документации.

ПолеТипОписание
slugstringУникальный идентификатор платежного счёта.

web_app_setup_back_button​

Обновляет настройки Кнопки Назад.

ПолеТипОписание
is_visiblebooleanОтобразить кнопку.

web_app_set_background_color​

Обновляет фоновый цвет приложения TWA.

ПолеТипОписание
colorstringЦвет в формате #RRGGBB.

web_app_set_header_color​

Обновляет цвет шапки приложения TWA.

ПолеТипОписание
color_keystringКлюч цвета. Значения: bg_color или secondary_bg_color.

web_app_trigger_haptic_feedback​

Создаёт событие тактильного отклика.

ПолеТипОписание
typestring

Тип события тактильного отклика. Значения:

  • impact
  • notification
  • selection_change
impact_stylestring

Обязателен в случае, когда type равен impact. Значения:

  • light, сигнализирует о соприкосновении малых объектов UI
  • medium, сигнализирует о соприкосновении средних объектов UI
  • heavy, сигнализирует о соприкосновении больших объектов UI
  • rigid, сигнализирует о соприкосновении тяжелых или негибких объектов UI
  • soft, сигнализирует о соприкосновении лёгких или гибких объектов UI
notification_typestring

Обязателен в случае, когда type является notification. Значения:

  • error, сообщает о том, что событие провалилось
  • success, сообщает о том, что событие успешно выполнилось
  • warning, сообщает о том, что событие произвело предупреждение

v6.2​

web_app_open_popup​

Открывает новое всплывающее окно. В случае его закрытия, будет произведено событие popup_closed.

ПолеТипОписание
titlestringТекст в заголовке. От 0 до 64 символов.
messagestringТекст в основном блоке. От 1 до 256 символов.
buttonsPopupButton[]Список кнопок. От 1 до 3 кнопок.
PopupButton​
ПолеТипОписание
idstringИдентификатор кнопки. От 0 до 64 символов.
typestring

Тип кнопки. Значения:

  • default, кнопка со стандартным стилем
  • destructive, кнопка, которая указывает на деструктивное действие (например, "Удалить")
  • ok, кнопка с локализованным текстом "OK"
  • close, кнопка с локализованным текстом "Закрыть"
  • cancel, кнопка с локализованным текстом "Отменить"
textstringТекст внутри кнопки. От 0 до 64 символов. Игнорируется  когда type равен ok, closeили cancel.

v6.4​

web_app_open_scan_qr_popup​

Открывает сканнер QR-кодов. В случае его закрытия будет создано событие scan_qr_popup_closed. Если сканнеру удалось просканировать QR-код, будет создано событие qr_text_received.

ПолеТипОписание
textstringОпциональное. Текст для отображения в QR-сканнере.

web_app_close_scan_qr_popup​

Закрывает QR-сканнер. В качестве результата будет создано событие scan_qr_popup_closed.

web_app_read_text_from_clipboard​

Производит чтение текста из буфера обмена. Метод принимает идентификатор запроса, который используется для корректного получения события qr_text_received.

ПолеТипОписание
req_idstringЛокальный идентификатор вызова метода. Возвращается в событии qr_text_received.
